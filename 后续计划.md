# 后续执行计划

承接 [拆分规则.md]。现在 `tables/` 已是「一表一文件」，但为了自包含，存在大量重复代码（加载样板、超窗/缺失检测、AE 预处理、两层表头 Excel 写法、汇总表重算）。本计划的目标是：**在不改变任何输出的前提下，把重复收敛进 utils 层**，让每个表文件变薄、且 fork 到新项目时改动集中。

## 0. 已锁定的方向（讨论结论，勿推翻）
- **一表一文件**：保留，复用单元是"表"不是"章节"。
- **只做项目内 DRY，不做跨项目通用框架**：当前只有一个项目、且项目间输入输出都不一致，任何"通用引擎/配置驱动"都是投机（参考 `utils/subject_status.py` 那套已脱节、无人使用的 config 引擎，应视为死代码）。
- **用函数 + 模块，不用类、不用 YAML 规则 DSL**：纯 df→df 变换，模块即命名空间。
- **顺序**：先抽 utils，再让表文件调用 utils。反过来会得到一堆样板。

## 1. 指导用的"八步操作模型"
每张表大致由以下操作组成（顺序可重复、可交错）。它只作**约定 + 决定什么进 utils**，不做成运行时引擎：

| 组 | 操作 | 复用性 / 去向 |
|---|---|---|
| 固定书挡（前） | 1 读取 | 高 → `loaders.py` |
| | 2 归一化（列名映射、日期/数值 parse、多表 concat、去重） | 高 → `loaders.py` / `builders.py` |
| 柔性内核（每表自由组合） | 3 筛选（布尔过滤、组内选行、去重留首/末） | 低，留在表文件 |
| | 4 变形（melt/pivot、groupby 聚合、crosstab） | 中，聚合可进 `builders.py` |
| | 5 派生（日期差、np.where、多选拼接、regex） | 低，留在表文件 |
| | 6 连接（富化连接=高复用；关联/自连接=独有） | 富化 → `loaders.py`，关联留表文件 |
| 固定书挡（后） | 7 格式化（选列、列序、改展示名、strftime、%/Int、加 No.、算 lc/ls） | 高 → utils |
| | 8 输出（三线表/xlsx/两层表头、标题含例次例数、脚注、合并列） | 高 → `output_format.py` |

要点：**1/2/7/8 是固定书挡（进 utils），3-6 是柔性内核（留在每张表文件，必要时调 builder）**。

## 2. 目标目录结构
```
utils/
  columns.py        # 【新】sheet 名 / 列名 / 访视名 常量 —— fork 新项目时的主要改动入口
  loaders.py        # 【新】薄加载层（写死本试验 schema，不参数化）
  builders.py       # 【新】多张表共享的中间计算
  output_format.py  # 【扩充】把 f.py 的两层表头 Excel 写法收编进来
  （subject_status.py / summarize_excel.py：评估后删除或归档）
tables/
  *.py              # 每个文件只 import utils + env，单表、可独立运行
```

## 3. 阶段拆解

### 阶段一：抽 `loaders.py`（消除加载样板）
把反复出现的读取收成小函数（**写死本项目 schema**，不为别的试验做通用化）：
- `load_rand(cols=…)` —— DS_RAND（受试者/随机号/随机日期/受试者状态…），各表按需取列
- `load_completion()` —— DS_END 过滤"试验完成情况总结" → 是否完成试验
- `load_dose_dates()` —— EC 首/末次用药日期 + 治疗天数（出现 ≥6 次的那段 groupby）
- `load_first_dose()` —— EC 单受试者最早服药日期（f.py 给药前后分组用）
- `load_assessments_long(sheets, date_col_map)` —— d/e 把多张 sheet 归一化堆成"以评估日期为键的长表"
- **验证**：先只抽 `load_rand()`，替换全部调用点，逐表比对输出无变化，再抽下一个。

### 阶段二：抽 `builders.py` + 扩 `output_format.py`（消除算法/输出重复）
- `build_window_table(类别, sheets)` —— d.py 超窗检测（随机日期±时间窗→判超窗）。`安全性/其他/汇总` 不再各自重算 `visit_table`，改为调用。
- `detect_missing(...)` —— e.py "是否评估=是但结果空 / =否则整表缺失"模式。
- `pair_pre_post(df, has_unit, has_desc)` —— f.py 给药前后 `pick_rows` 配对（4 份合一）。
- `output_format.py` 增 `export_two_level_header(...)` —— f.py 那 4 份两层表头 Excel 写法收编为一个函数（参数化固定列 / 前后段列）。
- **效果**：`超窗情况汇总` / `缺失情况汇总` / `用药后…整体情况` 这三个最臃肿的重算文件会大幅缩短。

### 阶段三：表文件迁移到 utils + 配置 jupytext
- 逐表把 §4 的样板替换成 `from utils.loaders import …`、`from utils.builders import …`。
- 为 `tables/` 配置 jupytext 配对（仿 `scripts/` 的 `notebooks//ipynb,../tables//py:percent`），并把 `.claude/settings.json` 的自动同步 hook 扩到 `tables/`。
- 视需要加一个 `run_all` 入口或主 notebook 串起 47 张表 + `下载.py` 打包。

## 4. 验证策略（贯穿全程）
**唯一验收标准：迁移前后产出的 docx/xlsx 内容一致。**
1. 在干净数据上先用原 `scripts/` 跑一遍，留存 `output/` 作为基线。
2. 每抽一个 utils 函数 / 迁移一张表，用 `tables/` 重跑该表，与基线**逐文件比对**（文件名、行数、列、例次例数、关键数值）。
3. 不一致就停下排查——优先怀疑 §拆分规则 4.2/4.3 的悬挂引用修复或重算是否引入偏差。
