# 报表脚本拆分规则（scripts/ → tables/）

本文档记录将 `scripts/[a-j].*.py` 拆成「一张表一个文件」（`tables/`）时遵循的规则，供日后复用 / 复核 / 套用到新项目。

> 本次拆分是**纯机械拆分**：逐字复制源代码，不抽函数、不优化、不改逻辑、不改格式。
> 唯一允许的改动，是「使每个文件能独立运行」所必需的最小调整（见 §4）。utils 抽取留到 [后续计划.md]。

## 1. 文件粒度
- **一张输出表 = 一个文件**。"一张表"的判定：以一次 `save_table_to_docx_threeline(...)` / `export_to_excel_with_format(...)` / 自定义 `pd.ExcelWriter` 写出为终点。
- 一个代码块内循环产出多个文件（如疗效超窗按访视分组导出多个 docx）算**同一张表**，放一个文件。

## 2. 文件命名
- 文件名 = 表的中文名，**去掉"表N"序号编号**。例：`表1 首末例受试者情况` → `首末例受试者情况.py`。
- **为什么不带序号**：源脚本里序号本身就不唯一/不连续——例如 `表34` 在 `a.py`（完成试验受试者清单）和 `d.py`（安全性评价指标超窗清单）各出现一次。用表名而非序号，正好避开这种冲突，也利于按"表"跨项目复用。
- **命名例外**（表名本身重复，必须加后缀区分）：
  - `f.py` 四张同名"用药后检查异常有临床意义清单"→ 用类别前缀：`生命体征…` / `体格检查…` / `12导联心电图…` / `实验室检查…`。
  - `i.py` 两个同名 WIP 块"方案禁止使用的合并药物或治疗"→ `…_表7.py` / `…_表8.py`（这两张导出被注释，属未完成，见 §5）。

## 3. 文件格式
- jupytext **percent** 格式。首个 cell 固定：
  ```python
  # %%
  # %run ../env.py
  ```
- 后续以 `# %%` 分隔 cell；保留该表相关的 `# %% [markdown]` 注释 cell（含原有的"待确认问题"文字）。
- `tables/` 与 `scripts/` 平级，`# %run ../env.py` 解析到项目根 `env.py`，路径仍正确。
- 新文件夹**暂未配置 jupytext 配对**（见 [后续计划.md] 阶段三）。

## 4. 自包含规则（关键）
每个文件必须能独立 `%run` 跑通，因此：

### 4.1 复制共享的"顶部加载"
源脚本顶部的共享变量（`RAND`、`DS_END`、`SV_N`、`index`、`TW`、`sig` 等）若被该表使用，**原样复制**进该文件。这会造成跨文件重复——是预期的，待 utils 抽出后收敛。

### 4.2 修复"跨 cell 残留全局变量"（悬挂引用）
源脚本是 notebook 顺序执行，存在"用到前一张表残留变量"的情况。拆开后这些引用会失效，按**行为完全一致**的原则就地修复，已修复项：

| 文件 | 原问题 | 修复方式 |
|---|---|---|
| `完成试验受试者清单.py` | 用了上一表残留的 `end[col1]` | 改用本块同样读取的 `END["试验完成日期"]`（同一次读取，结果一致） |
| `因与试验用药品相关的不良事件而发生的提前终止治疗.py` | `measure_cols` 在使用处之后才定义 | 在使用前补 `measure_cols=[…6列…]`；并删除 save 之后只 `display(AE)` 的探索性尾部代码 |
| `受试者层面方案偏离情况.py`、`严重方案偏离情况.py` | 用了"中心层面"块定义的 `notes` | 各补 `notes = []` |
| `用药依从性汇总清单.py` | 依赖"违背清单(表26)"产出的 `df` | 文件内先**内联重算表26的 df**，再做汇总（不重复导出表26） |

### 4.3 重算"被多表共享的中间结果"
汇总类 / 去重类的表依赖兄弟表的中间 DataFrame。拆开后在文件内**原样重算**这些中间结果（重复但忠实，待 builders 抽出后收敛）：

| 文件 | 需重算的中间结果 |
|---|---|
| `安全性评价指标超窗清单.py`、`其他指标超窗清单.py` | 先重算"访视超窗"得 `visit_table`，用于 `visit_drop` 剔除已计入访视超窗的行 |
| `超窗情况汇总.py` | 重算 访视/疗效/安全性/其他 四类检测得 `visit_table/efficacy_table/safe_table/oth_table` 再聚合 |
| `缺失情况汇总.py` | 重算四类缺失检测 + 各自 `*_summary` 再汇总 |
| `用药后检查异常有临床意义整体情况.py` | 重算 VS/PE/EG/LB 四类配对结果（含 `temp_id_visit`）再汇总 |
| `与试验药物有关/无关/特别关注不良事件清单.py` | 三张表各自完整复制同一段 AE 预处理 |

### 4.4 删除无产出的探索代码
仅用于 `display`、不产生任何已保存文件的尾部探索代码，删除不带入（目前仅 `a.py` 表9 之后一段）。

## 5. 其他约定
- 源脚本里**被注释掉的导出**（如 `i.py` 表7/表8 的 `save_…`）**原样保留注释状态**，标记为 WIP。
- `scripts/下载.py` 是打包脚本（zip output），**不是表**，未拆分，保留在 `scripts/`。
- 不修改 `scripts/` 原文件、不触碰任何 `.ipynb`（原 `scripts/` 作为对照保留）。
